OlayoutCSSでは、"olayout.config.cjs" の [screens](../configuration#screens) のサイズに合わせて、ブレイクポイントごとにレイアウトが可能です。

各Olayoutの属性に値を指定する際、ブレイクポイント名と `:` の後に値を指定れば、マークアップしながらレスポンシブコーディングが出来ます。

| ブレイクポイント名 | デフォルトの最小幅 |
| ------------------ | ------------------ |
| `xxs`              | `375px`            |
| `xs`               | `414px`            |
| `sm`               | `576px`            |
| `md`               | `768px`            |
| `lg`               | `1024px`           |
| `xl`               | `1280px`           |
| `xxl`              | `1440px`           |


## マークアップでレスポンシブ

例えば、基本の文字サイズは `small`（14px）だが、ウインドウ幅 `768px` 以上で `normal`（16px）、`1024px`以上で `large`（18px）にするには以下のようになります。

```html "md:normal" "lg:large"
// index.html
<p data-font="small md:normal lg:large">Hello Olayout.</p>
```

これは、値を設定できるOlayoutのカスタムデータ属性であればどれでも利用できます。


---

## カスタムメディアクエリ

Olayout CSSはマークアップをしながらレイアウトを行うことにフォーカスしています。
従って、コンポーネントに関しては、ご自身のCSSファイルを使ってスタイリングをして頂く必要があります。

ただし、PostCSS版のOlayoutCSSを利用している場合は、あなたのCSSファイルの中で、"olayout.config.cjs" の [screens](../configuration#screens) のサイズごとのカスタムメディアクエリが利用できます。

使い方は簡単です。

`@media ()` の `()` 内に、`--ol-` プレフィックスとブレイクポイント名を指定するだけです。

```css
// style.css
@media (--ol-sm) {
 /* デフォルトでは screen and (min-width: 576px) と同じ */
}

@media (--ol-md) {
 /* デフォルトでは screen and (min-width: 768px) と同じ */
}

@media (--ol-lg) {
 /* デフォルトでは screen and (min-width: 1024px) と同じ */
}
```

これを利用することで、SassなどのCSSプリプロセッサを使わずとも、ブレイクポイントの管理が可能になります。


:::caution
CDN版では利用できません。
:::

## ブレイクポイントのカスタマイズ

"olayout.config.cjs" を使えば、ブレイクポイントのカスタマイズが可能です。

詳細については、[ブレークポイントのカスタマイズに関するドキュメント](../configuration#screens)を参照してください。
